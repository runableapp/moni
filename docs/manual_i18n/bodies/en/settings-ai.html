Settings → AI

Main → Chat uses the provider selected here. Local-first options (llama.cpp, Ollama) keep your journal and messages on your machine. Cloud providers send chat to their APIs — use them only if you accept that trade-off. Choose None to disable chat until you are ready.

Local AI (llama.cpp)

Moni can download a llama-server binary and a GGUF model, then manage the server process on 127.0.0.1 (ports 8081–8299). Recommended first-time flow: click Set up local AI or download binary + model separately, then Start server.

Key controls

  • Model preset — pick a bundled preset or choose another GGUF file. Changing model while the server runs may prompt to restart.
  • Use GPU if available — requires a GPU-capable llama-server build; pick a specific GPU device or “All GPUs”.
  • Disable model reasoning — recommended for chat; passes --reasoning off so replies appear in the panel (needed for Qwen 3 and similar “thinking” models).
  • Router mode — swap models without full restart when using a multi-model --models-dir layout.
  • Auto-start local AI on launch — starts the managed server after the app loads (binary and model must already be installed).
  • Chat reply length / timeout — cap tokens and wait time for slow hardware.
  • Binary and model directories — override default paths; Reset paths to defaults restores shipped layout.
Moni-managed server binds the first free port on localhost from 8081 to 8299. The port is saved with the process identity so Moni reconnects after restart.

Ollama

Point Moni at your running Ollama instance (default base URL on localhost). Select a model from the list Ollama exposes. Ensure Ollama is installed and the model is pulled (ollama pull …) before chatting.

Cloud providers (OpenAI, Claude, Gemini)

Enter your API key (stored in local config). Pick model and any provider-specific options shown in the panel. Cloud chat does not require a local GPU but sends conversation content to the vendor’s API — do not use for journals you cannot share externally.

Related hledger settings (under Settings → hledger): Verify hledger commands in AI replies lint assistant examples against the version-matched complist and your binary before you click Run.

AI Wrapper (experimental)

Connects to the Runable AI Wrapper desktop app on the same machine. Register a client in AI Wrapper → API, enable services in Preferences, then paste the registration token in Moni. Choose service, mode (Electron in-app webview or Chrome extension), and client label. Click Connect to list available services.

Contact Runable.app for AI Wrapper access if you do not have the companion app installed.

hledger docs (RAG)

Moni can embed hledger documentation so local AI answers cite correct flags and concepts. Status lines in the AI panel show:

  • Not available — optional for local AI; cloud may still work without it.
  • Found on disk; loads on first chat — doc index present, loads when you first send a message.
  • Ready — N doc chunks — retrieval is active for grounded replies.

For paths, search order, and complist details, check Settings → Logs → Debug in app.log, or the provider 📜 Logs button in the AI panel.

Chat behavior on Main

With a provider connected, type in Main → Chat. Moni may suggest runnable hledger commands; press Enter or use controls to execute them in the Ledger panel. With autocomplete enabled (Settings → hledger), typing hledger argv shows suggestion popover; Ctrl+H runs hledger from the input line.

Interaction logging (Settings → Logs) can record user input, AI conversation, and ledger panel results for your own audit trail.

Troubleshooting

  • Server not reachable — for llama.cpp, confirm Start server succeeded; open server log from the diagnostics LED. Check firewall is not blocking localhost.
  • Wrong GPU build — CPU-only binary cannot use GPU toggle; download or install a GPU-capable llama-server variant.
  • Empty or “thinking” replies — enable Disable model reasoning and restart the server.
  • Cloud errors — verify API key, model name, and network; read 📜 Logs for HTTP status.
  • RAG not loading — see app.log debug lines; first chat may take longer while the index loads.

Copy the llama.cpp diagnostics bundle (status + log tail) when reporting issues — it includes everything support needs without exposing your journal.
