カスタムボタン (YAML)

ツールバーと台帳ツールはYAML レシピで定義されます。設定 → ボタン の各カスタムボタンは、yamlRef（レシピ id またはファイルパス）でレシピを参照します。Moni は YAML を読み込み、hledger または外部コマンドを実行し、その結果を 台帳 パネルに描画します（表または等幅テキスト）。

レシピの保存場所

場所用途
~/.config/moni/custom_buttons/<id>.yamlユーザー上書き（id 検索時の最優先）
<executable>/custom_buttons/<id>.yamlMoni バイナリの隣に同梱
.yaml / .yml で終わる、または / を含む任意のパス参照時に直接読み込み

設定 → ボタン では、各行に yamlRef があります。レシピ id（例: journal_check）または絶対/相対パス（例: ~/recipes/my_tool.yaml）です。レシピ id は ^[a-zA-Z0-9][a-zA-Z0-9._-]*$ に一致する必要があります（.. やスラッシュは不可）。

同梱サンプルはアプリの custom_buttons/ フォルダにあります。Moni を再ビルドせずにカスタマイズしたい場合は、そのうち 1 つを設定ディレクトリにコピーしてください。

最小レシピ: 1 つの hledger コマンド

hledger check は journal に問題がない場合、何も出力しません。空のパネルではなく、わかりやすいメッセージを表示できます:

id: journal_check

meta:
  name: Journal check
  description: Run hledger check on the full journal.

run:
  mode: hledger
  argv:
    - check

output:
  plain_text: true
  when_empty: "Everything is ok! No issues reported by hledger check."
  • run.mode: hledger — Moni は 設定 → hledger の設定を使って hledger -f <your journal> … を実行します。
  • run.argv — フルのシェルコマンドではなく、-f <journal> の後ろに続く引数です。
  • output.plain_text: true — stdout/stderr を等幅テキストとして表示します。
  • output.when_empty — 表示結果が空のとき、この文字列を代わりに表示します。

行指向の出力を装飾付きテーブルに変換する

hledger stats は key: value 形式の行を出力します。これを 2 列の表に変換できます:

id: journal_stats

meta:
  name: Journal overview
  description: hledger stats summary.

run:
  mode: hledger
  argv:
    - stats

output:
  delimited_text:
    delimiter: ":"
    hide_header: true
    no_striping: true
    column_align: [right, left]
    column_widths: [30%, 1fr]
    column_borders: true
    column_styles:
      - background: accent-24
        bold: true
      - {}

output.delimited_text は、空でない各行を、最初に現れる delimiter の位置で分割します。columns を設定しない場合、既定の列見出しは Field / Value です。

マルチステップレシピ: hledger を実行し、CSV を解析して、表を表示する

多くの台帳ツールは steps を使います。各ステップには id があり、hledger を実行し、必要に応じて結果を解析または変換します。

id: register_recent

meta:
  name: Recent activity
  description: Register for a date range.

ui:
  modal:
    when_empty:
      period: ytd
    fields:
      - id: date_from
        type: date
        label_key: common.dateFrom
        optional: true
      - id: date_to
        type: date
        label_key: common.dateTo
        optional: true

steps:
  - id: reg
    run:
      argv:
        - register
        - -p
        - "{{period}}"
        - -O
        - csv
    parse:
      kind: hledger_register_csv

output:
  from_step: reg
  table_view: register

流れ: ユーザーがボタンをクリック → モーダルで日付を入力 → Moni が {{period}} を組み立てる → ステップが hledger … register -p <period> -O csv を実行 → パーサーが列を構築 → output.from_step でその表を選択 → table_view: register でレジスター風のドリルダウンを有効化。

モーダルを空のままにしたときの既定日付

ui:
  modal:
    when_empty:
      period: lastyear
    fields:
      - id: date_from
        type: date
        optional: true
      - id: date_to
        type: date
        optional: true

レシピが {{period}} を使っていて、ユーザーが 2 つの日付をどちらも空欄にした場合は when_empty.period が適用されます。ユーザーが入力した日付が常に優先されます。

トークン範囲
lastyear前の暦年
thisyear現在の暦年
thismonth月初 .. 今日
lastmonth前の暦月
last30days今日 − 30 日 .. 今日
ytd, yeartodate1 月 1 日 .. 今日

変換: balance CSV を整形する

よくあるパターンは次の通りです: run → parse hledger_balance_csv → transform → output。

steps:
  - id: csv
    run:
      argv:
        - balance
        - "^expenses:"
        - -p
        - "{{period}}"
        - --flat
        - -O
        - csv
    parse:
      kind: hledger_balance_csv
  - id: table
    transform:
      kind: flat_balance_table
      source_step: csv

output:
  from_step: table

merge_account_profiles は口座名を 設定 の表示名プロファイルに従って変換します。stacked_balance_reports は 2 つの balance CSV ステップを、セクション見出し付きの 2 列レポート 1 つにまとめます。

1 つの結果に複数のテーブルを表示する

分離されたセクション（例: バランスシート）:

output:
  sections:
    - title: Assets
      from_step: assets_table
    - title: Liabilities
      from_step: liab_table

結合比較（口座ごとに 1 行、期間ごとに 1 列）:

output:
  merge_balance_sections: true
  sections:
    - title: "{{period_a}}"
      from_step: table_a
    - title: "{{period_b}}"
      from_step: table_b

セクションに when_empty を追加すると、そのセクションに行がない場合にプレースホルダーメッセージを表示できます。

外部プログラム（hledger 以外）

実行ファイル

argv[0] がプログラムで、残りが引数です。モーダルの値は実行前に argv に置換されます。

run:
  mode: executable
  argv:
    - /usr/bin/cal
    - "{{month}}"
    - "{{year}}"

output:
  plain_text: true

シェル

スクリプト全体を 1 つの文字列として指定します（Unix: sh -c、Windows: cmd /C）:

run:
  mode: shell
  script: |
    echo "Journal: $HLEDGER_JOURNAL"
    wc -l "$HLEDGER_JOURNAL"

環境には HLEDGER_JOURNAL、HLEDGER_JOURNAL_FILE、MONI_TOOLS_DIR が含まれます。

出力の後処理: パイプと空メッセージ

output:
  plain_text: true
  pipe:
    argv: ["grep", "-i", "error"]

output.when_empty はパイプ処理後の結果全体に適用されます。マルチステップの表で行が 0 件の場合、空のグリッドではなく指定したメッセージが表示されます。

argv 内のプレースホルダーを置換した後、空の -b、-e、-p、-f とその値は削除されるため、未使用の期間/ファイルフラグは自動的に消えます。

組み込みクエリヘルパー

YAML に次のプレースホルダーが含まれている場合、Moni が自動で補います:

プレースホルダー空の場合の値
{{expense_query}}^expenses またはモーダル expense_leaf に基づく expenses:<leaf>
{{balance_query}}^income またはモーダル customer に基づく income:consulting:<customer>
{{tag_slice_query}}^expenses: またはモーダル tag に基づく expenses tag:<tag>

任意のチャート（pie, bar, line）

モーダルにチャート用フィールドを追加し、YAML の output.chart を設定します。標準のモーダルフィールドは次の通りです:

ui:
  modal:
    when_empty:
      chart_display: "off"
      chart_period: monthly
    fields:
      - id: chart_display
        type: select
        optional: true
        options: ["off", "pie", "bar", "line"]
      - id: chart_period
        type: select
        optional: true
        options: [daily, monthly, yearly]

output:
  chart:
    param: chart_display
    period:
      param: chart_period
      argv_param: period_flag
      when_kinds: [bar, line]
      flags: { daily: "-D", monthly: "-M", yearly: "-Y" }
    layouts:
      bar: stacked_period
      line: period_series
      pie: category_totals

設定 で接続する

  1. レシピを ~/.config/moni/custom_buttons/my_tool.yaml として保存し、トップレベルの id: を一致させます。
  2. 設定 → ボタン を開き、行を追加または編集します。
  3. yamlRef に id（例: my_tool）またはフルパスを設定します。
  4. ボタンを有効にして メイン に戻ると、ツールバーに表示されます。
アプリに同梱されたレシピはよい出発点です。1 つコピーして設定フォルダで編集し、yamlRef をそのコピーに向けてください。
