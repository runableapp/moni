タブバー

ウィンドウ上部には 7 つのタブが並んでいます。日々の台帳作業、口座の閲覧、レポート、CSV インポート、設定、アプリ内ドキュメントをまとめて扱えます。

タブ用途
🏠 メイン台帳 パネル + AI チャット。ツールバーのボタンで hledger ツールを実行
🏦 口座残高チャートとレジスターを表示する口座を選択
🧾 取引先取引先を一覧し、取引先レジスタを開く
📈 レポートより長い要約ビュー（月次レポートなど）
📥 インポート銀行の CSV エクスポートをドロップし、hledger import でプレビュー
⚙️ 設定hledger のパス、AI、ボタン、テーマ、バックアップなど
❓ ヘルプアプリに同梱された組み込み Markdown トピック

メイン

メイン タブではウィンドウが 2 つのパネルに分かれます。台帳（左側）には、ツールバーボタンや組み込みツールの結果が表示され、表、チャート、プレーンテキスト出力が積み重なります。クリア を使うとスタックをリセットできます。チャット（右側）は 設定 → AI で選択した AI プロバイダーと会話します。何も選択されていない場合は、Moni が AI の有効化方法を案内します。

ツールバーボタンは YAML レシピです（カスタムボタン参照）。hledger のレポート、レジスター、バランスシート、外部スクリプトをワンクリックで実行できます。チャット は hledger コマンドを提案でき、任意の検証機能で、インストール済みの hledger バージョンとの整合性を確認できます。

口座

一覧から口座を選択します（設定 → 口座 で設定）。Moni は残高推移チャートと、その口座で絞り込んだレジスターを表示します。口座の表示名、アイコン、金融機関のヒントは 設定 の口座プロファイルから取得されます。

取引先

journal から抽出された取引先を閲覧できます。取引先を選ぶと、その取引先のレジスターをモーダルダイアログで開けます。特定の店舗や顧客に対する支出を確認したいときに便利です。

レポート

レポート は、より時間のかかる処理や要約表示のための専用エリアです。たとえば月次の損益レポートのようなビューを想定しています。ここは今後のリリースで拡張されますが、日常的なレポートの多くは現時点では メイン のツールバーボタンにあります。

インポート

インポート を開き、.csv、.ssv、.tsv ファイルをドロップゾーンに置くか、インポート ▾ メニューを使います。Moni はインポートルールを適用し、journal に書き込む前に必ず hledger import --dry-run でプレビューします。

利用できるワークフローは 2 つあります。classic import（ファイル全体のインポートと重複追跡用 .latest）と、スマートインポート（行を選択し、選んだ journal テキストだけを追記）です。スマートインポート は 設定 → CSV インポート で切り替えます。インポート全体のガイドは、アプリ内の ヘルプ → インポート を開いてください。

設定

設定 は左側のセクション一覧で構成されています。一般、AI、口座、MCP、詳細設定、AI エージェント、ボタン、外観、バックアップ、hledger、インポート、ログ です。まずは hledger（バイナリ + journal パス）と 口座（メイン と 口座 に表示する口座）を設定してください。

各セクションの詳細は専用の 設定 マニュアルを参照してください。AI の設定は 設定 → AI で扱っています。

ヘルプ

ヘルプ タブにはアプリ同梱の Markdown トピックが含まれています。既定では英語ですが、ドイツ語、スペイン語、フランス語、日本語、韓国語、中国語のローカライズ版も用意されています。トピックにはインポートワークフロー、AI の使い方、トラブルシューティングが含まれます。

ヒント: 大量インポートや、AI が提案した書き換えを試す前に journal をバックアップしておきましょう。自動バックアップは 設定 → バックアップ で有効にできます。
