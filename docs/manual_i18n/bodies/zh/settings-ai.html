设置 → AI

本地 AI (llama.cpp)

Moni 可以下载一个 llama-server 二进制文件和 GGUF 模型，然后在 127.0.0.1 上管理服务器进程（端口 8081–8299）。首次使用时，推荐流程是点击 设置本地 AI，或分别下载二进制文件和模型，然后点击 启动服务器。

关键控件

  • 使用的模型 — 选择内置预设，或改用其他 GGUF 文件。运行中切换模型时，可能会提示重启。
  • 可用时使用 GPU — 需要支持 GPU 的 llama-server 构建；可以选择特定 GPU 设备或“All GPUs”。
  • 禁用模型推理（reasoning） — 推荐用于聊天；会传递 --reasoning off，使回复能显示在面板中（Qwen 3 等“thinking”模型需要这样做）。
  • 不重启切换模型（路由模式） — 在使用多模型 --models-dir 布局时，无需完全重启即可切换模型。
  • 启动时自动启动本地 AI — 应用加载后自动启动受管服务器（二进制文件和模型必须已安装）。
  • 聊天 reply length / timeout — 为较慢硬件限制 token 数和等待时间。
  • 可执行文件目录 / 模型目录 — 覆盖默认路径；重置路径为默认值 可恢复为随应用提供的布局。
Moni 管理的服务器会绑定到 localhost 上 8081 到 8299 之间第一个可用端口。该端口会与进程标识一起保存，以便 Moni 在重启后重新连接。

Ollama

将 Moni 指向你正在运行的 Ollama 实例（默认 localhost base URL）。从 Ollama 暴露的列表中选择一个模型。聊天前请确保已安装 Ollama，并且模型已拉取（ollama pull …）。

云提供商 (OpenAI, Claude, Gemini)

输入你的 API key（保存在本地配置中）。选择面板中显示的模型以及各提供商特有的选项。云聊天不需要本地 GPU，但会将对话内容发送到供应商 API，因此不要将其用于不能对外共享的 journal。

相关 hledger 设置（位于 设置 → hledger 下）：验证 AI 回复中的 hledger 命令 会在你点击 Run 之前，使用与你版本匹配的 complist 和你的二进制文件检查助手给出的示例。

AI Wrapper（实验性）

连接到同一台机器上的 Runable AI Wrapper 桌面应用。在 AI Wrapper → API 中注册一个客户端，在 Preferences 中启用服务，然后将注册令牌粘贴到 Moni 中。选择服务、模式（Electron 应用内 webview 或 Chrome 扩展）以及客户端标签。点击 连接 可列出可用服务。

如果你尚未安装该配套应用，请联系 Runable.app 以获取 AI Wrapper 访问权限。

hledger 文档 (RAG)

Moni 可以嵌入 hledger 文档，使本地 AI 在回答时引用正确的标志和概念。AI 面板中的状态行会显示：

  • 不可用（本地 AI 可选）。 — 对本地 AI 来说是可选项；云提供商即使没有它也可能仍然可用。
  • 状态：文件在磁盘上；首次 AI 消息时加载到内存（任意提供商）。 — 文档索引已存在，首次发送消息时会加载。
  • 就绪 — N 个文档块。 — 已启用检索，可生成有依据的回复。

关于路径、搜索顺序和 complist 细节，请查看 设置 → 日志 → 调试 中的 app.log，或 AI 面板中提供商的 📜 日志 按钮。

主页 中的聊天行为

连接提供商后，在 主页 → 聊天 中输入内容。Moni 可能会建议可运行的 hledger 命令；按 Enter，或使用控件在 账本 面板中执行它们。启用自动补全（设置 → hledger）后，输入 hledger argv 时会显示建议弹窗；按 Ctrl+H 可直接从输入行运行 hledger。

交互日志（设置 → 日志）可记录用户输入、AI 对话以及账本面板结果，供你自己做审计追踪。

故障排查

  • OpenAI 兼容 API 无法访问。 — 对于 llama.cpp，请确认 启动服务器 已成功；可从诊断 LED 打开服务器日志。还要检查防火墙是否阻止 localhost。
  • Wrong GPU build — 纯 CPU 二进制文件无法使用 GPU 开关；请下载或安装支持 GPU 的 llama-server 变体。
  • Empty or “thinking” replies — 启用 禁用模型推理（reasoning） 并重启服务器。
  • Cloud errors — 检查 API key、模型名称和网络；通过 📜 日志 查看 HTTP 状态。
  • RAG not loading — 查看 app.log 调试行；首次聊天在加载索引时可能会更慢。

报告问题时，请复制 llama.cpp 的diagnostics bundle（状态 + 日志尾部）。它包含支持所需的全部信息，同时不会暴露你的 journal。
