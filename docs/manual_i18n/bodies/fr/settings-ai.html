Paramètres → IA

Accueil → Chat uses the provider selected here. Local-first options (llama.cpp, Ollama) keep your journal and messages on your machine. Cloud providers send chat to their APIs — use them only if you accept that trade-off. Choose Aucun to disable chat until you are ready.

Local IA (llama.cpp)

Moni can download a llama-server binary and a GGUF model, then manage the server process on 127.0.0.1 (ports 8081–8299). Recommended first-time flow: click Configurer l’IA locale or download binary + model separately, then Démarrer le serveur.

Key controls

  • Modèle à utiliser — pick a bundled preset or choose another GGUF file. Changing model while the server runs may prompt to restart.
  • Utiliser le GPU si disponible — requires a GPU-capable llama-server build; pick a specific GPU device or “All GPUs”.
  • Désactiver le raisonnement du modèle — recommended for chat; passes --reasoning off so replies appear in the panel (needed for Qwen 3 and similar “thinking” models).
  • Changer de modèle sans redémarrage (mode routeur) — swap models without full restart when using a multi-model --models-dir layout.
  • Démarrer l’IA locale au lancement — starts the managed server after the app loads (binary and model must already be installed).
  • Chat reply length / timeout — cap tokens and wait time for slow hardware.
  • Répertoire du binaire / Répertoire des modèles — override default paths; Réinitialiser les chemins par défaut restores shipped layout.
Moni-managed server binds the first free port on localhost from 8081 to 8299. The port is saved with the process identity so Moni reconnects after restart.

Ollama

Point Moni at your running Ollama instance (default base URL on localhost). Select a model from the list Ollama exposes. Ensure Ollama is installed and the model is pulled (ollama pull …) before chatting.

Cloud providers (OpenIA, Claude, Gemini)

Enter your API key (stored in local config). Pick model and any provider-specific options shown in the panel. Cloud chat does not require a local GPU but sends conversation content to the vendor’s API — do not use for journals you cannot share externally.

Related hledger settings (under Paramètres → hledger): Vérifier les commandes hledger dans les réponses IA lint assistant examples against the version-matched complist and your binary before you click Run.

IA Wrapper (experimental)

Connecters to the Runable IA Wrapper desktop app on the same machine. Register a client in IA Wrapper → API, enable services in Preferences, then paste the registration token in Moni. Choose service, mode (Electron in-app webview or Extension Chrome), and client label. Click Connecterer to list available services.

Contact Runable.app for IA Wrapper access if you do not have the companion app installed.

Index de documentation hledger (RAG)

Moni can embed hledger documentation so local IA answers cite correct flags and concepts. Status lines in the IA panel show:

  • Indisponible — optional for local IA; cloud may still work without it.
  • Sur disque ; charge au premier chat — doc index present, loads when you first send a message.
  • Prêt — N fragments de documents — retrieval is active for grounded replies.

For paths, search order, and complist details, check Paramètres → Journaux → Débogage in app.log, or the provider 📜 Journaux button in the IA panel.

Chat behavior on Accueil

With a provider connected, type in Accueil → Chat. Moni may suggest runnable hledger commands; press Enter or use controls to execute them in the Résultats du panneau comptable. With autocomplete enabled (Paramètres → hledger), typing hledger argv shows suggestion popover; Ctrl+H runs hledger from the input line.

Interaction logging (Paramètres → Journaux) can record user input, IA conversation, and ledger panel results for your own audit trail.

Troubleshooting

  • L'API compatible OpenIA n'est pas accessible. — for llama.cpp, confirm Démarrer le serveur succeeded; open server log from the diagnostics LED. Check firewall is not blocking localhost.
  • Wrong GPU build — CPU-only binary cannot use GPU toggle; download or install a GPU-capable llama-server variant.
  • Empty or “thinking” replies — enable Désactiver le raisonnement du modèle and restart the server.
  • Cloud errors — verify API key, model name, and network; read 📜 Journaux for HTTP status.
  • RAG not loading — see app.log debug lines; first chat may take longer while the index loads.

Copy the llama.cpp diagnostics bundle (status + log tail) when reporting issues — it includes everything support needs without exposing your journal.
