사용자 지정 버튼 (YAML)

도구 모음과 원장 도구는 YAML 레시피로 정의됩니다. 설정 → 버튼의 각 사용자 지정 버튼은 yamlRef(레시피 id 또는 파일 경로)로 레시피를 가리킵니다. Moni는 YAML을 불러와 hledger 또는 외부 명령을 실행하고, 결과를 원장 패널에 렌더링합니다(표 또는 고정폭 텍스트).

레시피 위치

위치용도
~/.config/moni/custom_buttons/<id>.yaml사용자 재정의(id 조회 시 최우선)
<executable>/custom_buttons/<id>.yamlMoni 바이너리 옆에 기본 포함
.yaml / .yml로 끝나거나 /를 포함하는 모든 경로참조되면 직접 로드

설정 → 버튼에서 각 행에는 yamlRef가 있습니다. 레시피 id(예: journal_check)이거나 절대/상대 경로(예: ~/recipes/my_tool.yaml)입니다. 레시피 id는 ^[a-zA-Z0-9][a-zA-Z0-9._-]*$를 충족해야 합니다(..나 슬래시 없음).

기본 제공 샘플은 앱의 custom_buttons/ 폴더에 있습니다. Moni를 다시 빌드하지 않고 사용자화하려면 하나를 설정 디렉터리로 복사하세요.

최소 레시피: hledger 명령 하나

hledger check는 저널이 정상일 때 아무 출력도 만들지 않습니다. 빈 패널 대신 친절한 메시지를 표시하세요:

id: journal_check

meta:
  name: Journal check
  description: Run hledger check on the full journal.

run:
  mode: hledger
  argv:
    - check

output:
  plain_text: true
  when_empty: "Everything is ok! No issues reported by hledger check."
  • run.mode: hledger — Moni는 설정 → hledger를 사용해 hledger -f <your journal> …를 실행합니다.
  • run.argv — 전체 셸 명령줄이 아니라 -f <journal> 뒤의 인자입니다.
  • output.plain_text: true — stdout/stderr를 고정폭 텍스트로 표시합니다.
  • output.when_empty — 표시할 출력이 비어 있으면 이 문자열을 대신 보여줍니다.

줄 단위 출력을 스타일이 적용된 표로 파싱하기

hledger stats는 key: value 형식의 줄을 출력합니다. 이를 두 열 표로 바꿉니다:

id: journal_stats

meta:
  name: Journal overview
  description: hledger stats summary.

run:
  mode: hledger
  argv:
    - stats

output:
  delimited_text:
    delimiter: ":"
    hide_header: true
    no_striping: true
    column_align: [right, left]
    column_widths: [30%, 1fr]
    column_borders: true
    column_styles:
      - background: accent-24
        bold: true
      - {}

output.delimited_text는 비어 있지 않은 각 줄을 delimiter가 처음 나타나는 위치에서 나눕니다. columns를 설정하지 않으면 기본 열 머리글은 Field / Value입니다.

다단계 레시피: hledger 실행, CSV 파싱, 표 표시

대부분의 원장 도구는 steps를 사용합니다. 각 단계는 id를 가지며, hledger를 실행하고, 필요하면 결과를 파싱하거나 변환합니다.

id: register_recent

meta:
  name: Recent activity
  description: Register for a date range.

ui:
  modal:
    when_empty:
      period: ytd
    fields:
      - id: date_from
        type: date
        label_key: common.dateFrom
        optional: true
      - id: date_to
        type: date
        label_key: common.dateTo
        optional: true

steps:
  - id: reg
    run:
      argv:
        - register
        - -p
        - "{{period}}"
        - -O
        - csv
    parse:
      kind: hledger_register_csv

output:
  from_step: reg
  table_view: register

흐름: 사용자가 버튼을 클릭 → 모달이 날짜를 수집 → Moni가 {{period}}를 구성 → 단계가 hledger … register -p <period> -O csv 실행 → 파서가 열 구성 → output.from_step가 해당 표 선택 → table_view: register가 등록부 스타일 드릴다운 활성화.

모달을 비워 둘 때의 기본 날짜

ui:
  modal:
    when_empty:
      period: lastyear
    fields:
      - id: date_from
        type: date
        optional: true
      - id: date_to
        type: date
        optional: true

레시피가 {{period}}를 사용하고 사용자가 두 날짜를 모두 비워 두면 when_empty.period가 적용됩니다. 사용자가 입력한 날짜가 항상 우선합니다.

토큰범위
lastyear이전 달력 연도
thisyear현재 달력 연도
thismonth이번 달 1일 .. 오늘
lastmonth이전 달력 월
last30days오늘 − 30일 .. 오늘
ytd, yeartodate1월 1일 .. 오늘

변환: balance CSV 재구성

흔한 패턴은 다음과 같습니다: run → parse hledger_balance_csv → transform → output.

steps:
  - id: csv
    run:
      argv:
        - balance
        - "^expenses:"
        - -p
        - "{{period}}"
        - --flat
        - -O
        - csv
    parse:
      kind: hledger_balance_csv
  - id: table
    transform:
      kind: flat_balance_table
      source_step: csv

output:
  from_step: table

merge_account_profiles는 계정 이름을 설정 프로필 표시 이름으로 매핑합니다. stacked_balance_reports는 두 개의 balance CSV 단계를 섹션 제목이 있는 2열 리포트 하나로 결합합니다.

한 결과에 여러 표 넣기

분리된 섹션(예: 대차대조표):

output:
  sections:
    - title: Assets
      from_step: assets_table
    - title: Liabilities
      from_step: liab_table

병합 비교(계정당 한 행, 기간당 한 열):

output:
  merge_balance_sections: true
  sections:
    - title: "{{period_a}}"
      from_step: table_a
    - title: "{{period_b}}"
      from_step: table_b

섹션에 when_empty를 추가하면 해당 섹션에 행이 없을 때 플레이스홀더 메시지를 표시할 수 있습니다.

외부 프로그램(hledger 아님)

실행 파일

argv[0]는 프로그램이고, 나머지 항목은 인자입니다. 실행 전에 모달 값이 argv에 치환됩니다.

run:
  mode: executable
  argv:
    - /usr/bin/cal
    - "{{month}}"
    - "{{year}}"

output:
  plain_text: true

셸

전체 스크립트를 하나의 문자열로 지정합니다(Unix: sh -c; Windows: cmd /C):

run:
  mode: shell
  script: |
    echo "Journal: $HLEDGER_JOURNAL"
    wc -l "$HLEDGER_JOURNAL"

환경에는 HLEDGER_JOURNAL, HLEDGER_JOURNAL_FILE, MONI_TOOLS_DIR가 포함됩니다.

출력 후처리: 파이프와 빈 메시지

output:
  plain_text: true
  pipe:
    argv: ["grep", "-i", "error"]

output.when_empty는 파이프 이후의 전체 결과에 적용됩니다. 여러 단계의 표에 행이 하나도 없으면 빈 그리드 대신 지정한 메시지를 보여줍니다.

argv의 플레이스홀더를 치환한 뒤, 비어 있는 -b, -e, -p, -f와 그 값은 제거되므로, 선택적 기간/파일 플래그는 사용하지 않을 때 사라집니다.

내장 쿼리 도우미

YAML에 다음 플레이스홀더가 있으면 Moni가 자동으로 채워 넣습니다:

플레이스홀더비어 있을 때 값
{{expense_query}}^expenses 또는 모달 expense_leaf의 expenses:<leaf>
{{balance_query}}^income 또는 모달 customer의 income:consulting:<customer>
{{tag_slice_query}}^expenses: 또는 모달 tag의 expenses tag:<tag>

선택형 차트(pie, bar, line)

모달에 차트 필드와 YAML의 output.chart를 추가하세요. 표준 모달 필드는 다음과 같습니다:

ui:
  modal:
    when_empty:
      chart_display: "off"
      chart_period: monthly
    fields:
      - id: chart_display
        type: select
        optional: true
        options: ["off", "pie", "bar", "line"]
      - id: chart_period
        type: select
        optional: true
        options: [daily, monthly, yearly]

output:
  chart:
    param: chart_display
    period:
      param: chart_period
      argv_param: period_flag
      when_kinds: [bar, line]
      flags: { daily: "-D", monthly: "-M", yearly: "-Y" }
    layouts:
      bar: stacked_period
      line: period_series
      pie: category_totals

설정에서 연결

  1. 레시피를 ~/.config/moni/custom_buttons/my_tool.yaml로 저장하고, 최상위 id:가 일치하도록 설정합니다.
  2. 설정 → 버튼를 열고 행을 추가하거나 편집합니다.
  3. yamlRef를 id(예: my_tool) 또는 전체 경로로 설정합니다.
  4. 버튼을 활성화한 뒤 메인으로 돌아가면 도구 모음에 나타납니다.
앱 설치에 포함된 기본 레시피는 좋은 출발점입니다. 하나를 설정 폴더로 복사해 편집하고, yamlRef가 그 복사본을 가리키게 하세요.
