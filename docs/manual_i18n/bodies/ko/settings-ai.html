로컬 AI (llama.cpp)

Moni는 llama-server 바이너리와 GGUF 모델을 다운로드한 뒤, 127.0.0.1에서 서버 프로세스(포트 8081–8299)를 관리할 수 있습니다. 처음 사용하는 경우 권장 순서는 로컬 AI 설정를 클릭하거나 바이너리와 모델을 각각 다운로드한 다음 서버 시작를 누르는 것입니다.

주요 제어

  • 사용할 모델 — 기본 제공 프리셋을 고르거나 다른 GGUF 파일을 선택합니다. 서버 실행 중 모델을 바꾸면 재시작을 요구할 수 있습니다.
  • 가능하면 GPU 사용 — GPU 지원 llama-server 빌드가 필요합니다. 특정 GPU 장치나 “All GPUs”를 선택할 수 있습니다.
  • 모델 추론(reasoning) 비활성화 — 채팅에 권장됩니다. --reasoning off를 전달해 답변이 패널에 표시되도록 합니다(Qwen 3 같은 “thinking” 모델에 필요).
  • 재시작 없이 모델 전환(라우터 모드) — 다중 모델 --models-dir 레이아웃을 사용할 때 전체 재시작 없이 모델을 전환합니다.
  • 실행 시 로컬 AI 자동 시작 — 앱이 로드된 뒤 관리형 서버를 시작합니다(바이너리와 모델이 이미 설치되어 있어야 함).
  • 채팅 reply length / timeout — 느린 하드웨어를 위해 토큰 수와 대기 시간을 제한합니다.
  • 실행 파일 디렉터리 / 모델 디렉터리 — 기본 경로를 재정의합니다. 경로를 기본값으로 재설정는 기본 레이아웃으로 되돌립니다.
Moni가 관리하는 서버는 localhost에서 8081부터 8299까지의 포트 중 처음 비어 있는 포트에 바인딩됩니다. 포트는 프로세스 식별 정보와 함께 저장되어 Moni가 재시작 후 다시 연결할 수 있습니다.

Ollama

실행 중인 Ollama 인스턴스(기본 localhost base URL)를 Moni에 연결하세요. Ollama가 제공하는 목록에서 모델을 선택합니다. 채팅하기 전에 Ollama가 설치되어 있고 모델이 가져와져 있는지(ollama pull …) 확인하세요.

클라우드 제공자 (OpenAI, Claude, Gemini)

API 키를 입력하세요(로컬 설정에 저장됨). 패널에 표시되는 모델과 제공자별 옵션을 선택할 수 있습니다. 클라우드 채팅은 로컬 GPU가 필요 없지만 대화 내용을 공급업체 API로 전송하므로, 외부와 공유할 수 없는 저널에는 사용하지 마세요.

관련 hledger 설정(설정 → hledger 아래): AI 응답의 hledger 명령 검증는 Run을 누르기 전에 설치된 바이너리와 버전에 맞는 complist를 기준으로 도우미 예시를 검사합니다.

AI Wrapper (실험적)

같은 컴퓨터에서 실행 중인 Runable AI Wrapper 데스크톱 앱에 연결합니다. AI Wrapper → API에서 클라이언트를 등록하고, Preferences에서 서비스를 활성화한 뒤, Moni에 등록 토큰을 붙여 넣으세요. 서비스, 모드(Electron 인앱 웹뷰 또는 Chrome 확장), 클라이언트 라벨을 선택합니다. 연결를 클릭하면 사용 가능한 서비스 목록을 불러옵니다.

보조 앱이 설치되어 있지 않다면 AI Wrapper 접근에 대해 Runable.app에 문의하세요.

hledger 문서 (RAG)

Moni는 hledger 문서를 임베딩하여 로컬 AI 답변이 올바른 플래그와 개념을 인용하도록 할 수 있습니다. AI 패널의 상태 줄에는 다음이 표시됩니다:

  • 사용 불가 — 로컬 AI에서는 선택 사항이며, 클라우드는 이것이 없어도 동작할 수 있습니다.
  • 디스크에 있음; 첫 채팅 시 로드 — 문서 인덱스가 있으며 첫 메시지를 보낼 때 로드됩니다.
  • 준비됨 — N개 문서 청크 — 근거 기반 답변을 위한 검색이 활성화되어 있습니다.

경로, 검색 순서, complist 세부 정보는 설정 → 로그 → 디버그의 app.log 또는 AI 패널의 제공자 📜 로그 버튼을 확인하세요.

메인의 채팅 동작

제공자가 연결되면 메인 → 채팅에 입력하세요. Moni는 실행 가능한 hledger 명령을 제안할 수 있으며, Enter를 누르거나 컨트롤을 사용해 원장 패널에서 실행할 수 있습니다. 자동완성(설정 → hledger)이 활성화되어 있으면 hledger argv 입력 시 제안 팝오버가 나타나며, Ctrl+H로 입력 줄에서 hledger를 실행합니다.

상호작용 로깅(설정 → 로그)은 사용자 입력, AI 대화, 원장 패널 결과를 개인 감사 기록용으로 저장할 수 있습니다.

문제 해결

  • OpenAI 호환 API에 연결할 수 없습니다. — llama.cpp의 경우 서버 시작가 성공했는지 확인하세요. 진단 LED에서 서버 로그를 열 수 있습니다. 방화벽이 localhost를 차단하지 않는지도 확인하세요.
  • Wrong GPU build — CPU 전용 바이너리는 GPU 토글을 사용할 수 없습니다. GPU 지원 llama-server 변형을 다운로드하거나 설치하세요.
  • Empty or “thinking” replies — 모델 추론(reasoning) 비활성화을 활성화하고 서버를 재시작하세요.
  • Cloud errors — API 키, 모델 이름, 네트워크를 확인하고, HTTP 상태는 📜 로그에서 읽으세요.
  • RAG not loading — app.log의 디버그 줄을 확인하세요. 인덱스를 로드하는 동안 첫 채팅은 더 오래 걸릴 수 있습니다.

문제를 보고할 때는 llama.cpp diagnostics bundle(상태 + 로그 끝부분)을 복사하세요. 저널을 노출하지 않으면서 지원팀에 필요한 모든 정보가 포함됩니다.
