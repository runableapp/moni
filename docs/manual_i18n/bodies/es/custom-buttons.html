Botones personalizados (YAML)

Las herramientas de barra y libro mayor se definen como YAML recipes. Each custom button in Configuración → Botones points at a recipe by yamlRef (recipe id or file path). Moni loads the YAML, runs hledger or external commands, y renders the result in the Resultados del panel del libro mayor (tables or monospace text).

Dónde viven las recetas

UbicaciónPropósito
~/.config/moni/custom_buttons/<id>.yamlSustituciones del usuario (máxima prioridad al buscar por id)
<executable>/custom_buttons/<id>.yamlIncluido junto al binario de Moni
Any path ending in .yaml / .yml, or containing /Se carga directamente al referenciarlo

En Configuración → Botones, cada fila tiene yamlRef: bien un id (p. ej. journal_check) o una ruta absoluta/relativa path (p. ej. ~/recipes/my_tool.yaml). Recipe ids must match ^[a-zA-Z0-9][a-zA-Z0-9._-]*$ (sin .. ni barras).

Los ejemplos incluidos están en la carpeta custom_buttons/ del app. Copia uno a tu directorio de configuración para personalizar sin recompilar Moni.

Receta mínima: un comando hledger

hledger check no produce salida cuando el diario está sano. Muestra un mensaje amigable en lugar de un panel vacío:

id: journal_check

meta:
  name: Journal check
  description: Run hledger check on the full journal.

run:
  mode: hledger
  argv:
    - check

output:
  plain_text: true
  when_empty: "Everything is ok! No issues reported by hledger check."
  • run.mode: hledger — Moni ejecuta hledger -f <your journal> … usando Configuración → hledger.
  • run.argv — argumentos tras -f <journal>, no la línea completa del shell.
  • output.plain_text: true — muestra stdout/stderr como texto de ancho fijo.
  • output.when_empty — si la salida mostrada está vacía, muestra esta cadena.

Convertir salida por líneas en una tabla con estilo

hledger stats imprime líneas key: value. Conviértelas en una tabla de dos columnas:

id: journal_stats

meta:
  name: Journal overview
  description: hledger stats summary.

run:
  mode: hledger
  argv:
    - stats

output:
  delimited_text:
    delimiter: ":"
    hide_header: true
    no_striping: true
    column_align: [right, left]
    column_widths: [30%, 1fr]
    column_borders: true
    column_styles:
      - background: accent-24
        bold: true
      - {}

output.delimited_text divide cada línea no vacía en la primera aparición de delimiter. Los encabezados por defecto son Field / Value salvo que definas columns.

Receta multipaso: ejecutar hledger, analizar CSV, mostrar tabla

La mayoría de herramientas usan steps: cada paso tiene un id, ejecuta hledger y opcionalmente analiza o transforma el resultado.

id: register_recent

meta:
  name: Recent activity
  description: Register for a date range.

ui:
  modal:
    when_empty:
      period: ytd
    fields:
      - id: date_from
        type: date
        label_key: common.dateFrom
        optional: true
      - id: date_to
        type: date
        label_key: common.dateTo
        optional: true

steps:
  - id: reg
    run:
      argv:
        - register
        - -p
        - "{{period}}"
        - -O
        - csv
    parse:
      kind: hledger_register_csv

output:
  from_step: reg
  table_view: register

Flujo: el usuario pulsa el botón → el modal recoge fechas → Moni construye {{period}} → el paso ejecuta hledger … register -p <period> -O csv → el analizador crea columnas → output.from_step selecciona esa tabla → table_view: register activa el detalle estilo registro.

Fechas por defecto cuando el modal queda en blanco

ui:
  modal:
    when_empty:
      period: lastyear
    fields:
      - id: date_from
        type: date
        optional: true
      - id: date_to
        type: date
        optional: true

Si la receta usa {{period}} y el usuario deja ambas fechas vacías, when_empty.period se aplica. Las fechas del usuario siempre prevalecen.

TokenIntervalo

thisyearAño calendario actual
thismonthPrimer día del mes .. hoy
lastmonthMes calendario anterior
last30daysHoy − 30 días .. hoy

Transformaciones: reformatear CSV de balance

Un patrón habitual: run → parse hledger_balance_csv → transform → output.

steps:
  - id: csv
    run:
      argv:
        - balance
        - "^expenses:"
        - -p
        - "{{period}}"
        - --flat
        - -O
        - csv
    parse:
      kind: hledger_balance_csv
  - id: table
    transform:
      kind: flat_balance_table
      source_step: csv

output:
  from_step: table

merge_account_profiles asigna nombres de cuenta a través de los nombres visibles del perfil en Configuración. stacked_balance_reports combina dos pasos CSV de balance en un informe de dos columnas con encabezados de sección.

Varias tablas en un resultado

Secciones separadas (p. ej. balance):

output:
  sections:
    - title: Assets
      from_step: assets_table
    - title: Liabilities
      from_step: liab_table

Comparación fusionada (una fila por cuenta, una columna por periodo):

output:
  merge_balance_sections: true
  sections:
    - title: "{{period_a}}"
      from_step: table_a
    - title: "{{period_b}}"
      from_step: table_b

Añade when_empty en una sección para un mensaje cuando no hay filas.

Programas externos (sin hledger)

Ejecutable

argv[0] es el programa; el resto son argumentos. Los valores del modal sustituyen argv antes de ejecutar.

run:
  mode: executable
  argv:
    - /usr/bin/cal
    - "{{month}}"
    - "{{year}}"

output:
  plain_text: true

Shell

Script completo como una cadena (Unix: sh -c; Windows: cmd /C):

run:
  mode: shell
  script: |
    echo "Journal: $HLEDGER_JOURNAL"
    wc -l "$HLEDGER_JOURNAL"

El entorno incluye HLEDGER_JOURNAL, HLEDGER_JOURNAL_FILE, y MONI_TOOLS_DIR.

Postprocesar salida: tubería y mensaje vacío

output:
  plain_text: true
  pipe:
    argv: ["grep", "-i", "error"]

output.when_empty se aplica al resultado completo tras la tubería. Para tablas multipaso sin filas, muestra tu mensaje en lugar de una cuadrícula vacía.

Los marcadores en argv se sustituyen; luego se omiten -b, -e, -p, -f and their values are dropped so optional period/file flags disappear when unused.

Ayudantes de consulta integrados

Si el YAML contiene estos marcadores, Moni los rellena automáticamente:

MarcadorSi está vacío, se convierte en
{{expense_query}}^expenses or expenses:<leaf> del modal expense_leaf
{{balance_query}}^income or income:consulting:<customer> del modal customer
{{tag_slice_query}}^expenses: or expenses tag:<tag> del modal tag

Gráficos opcionales (circular, barras, líneas)

Añade campos de gráfico al modal y output.chart en YAML. Campos estándar del modal:

ui:
  modal:
    when_empty:
      chart_display: "off"
      chart_period: monthly
    fields:
      - id: chart_display
        type: select
        optional: true
        options: ["off", "pie", "bar", "line"]
      - id: chart_period
        type: select
        optional: true
        options: [daily, monthly, yearly]

output:
  chart:
    param: chart_display
    period:
      param: chart_period
      argv_param: period_flag
      when_kinds: [bar, line]
      flags: { daily: "-D", monthly: "-M", yearly: "-Y" }
    layouts:
      bar: stacked_period
      line: period_series
      pie: category_totals

Conectar en Configuración

  1. Guarda tu receta como ~/.config/moni/custom_buttons/my_tool.yaml con un id:.
  2. Abre Configuración → Botones → añade o edita una fila.
  3. Establece yamlRef al id (p. ej. my_tool) o una ruta completa.
  4. Activa el botón y vuelve a Principal — aparece en la barra.
Las recetas incluidas en la instalación son un buen punto de partida: copia una a tu carpeta de configuración, edítala y apunta yamlRef a tu copia.
