ローカル AI (llama.cpp)

Moni は llama-server バイナリと GGUF モデルをダウンロードし、127.0.0.1 上のサーバープロセス（ポート 8081–8299）を管理できます。初回の推奨手順は、ローカル AI をセットアップ をクリックするか、バイナリとモデルを個別にダウンロードした後で サーバーを起動 を押すことです。

主なコントロール

  • 使用するモデル — 同梱プリセットを選ぶか、別の GGUF ファイルを指定します。サーバー実行中にモデルを変更すると、再起動を求められることがあります。
  • 利用可能な場合は GPU を使用 — GPU 対応の llama-server ビルドが必要です。特定の GPU デバイス、または “All GPUs” を選べます。
  • モデルの推論を無効化 — チャットでは推奨です。--reasoning off を渡して、返答がパネルに表示されるようにします（Qwen 3 のような “thinking” モデルで必要）。
  • 再起動なしでモデル切替（ルーターモード） — 複数モデルの --models-dir レイアウトを使う場合、完全再起動なしでモデルを切り替えます。
  • 起動時にローカル AI を自動開始 — アプリ読み込み後に管理対象サーバーを起動します（バイナリとモデルは事前にインストール済みである必要があります）。
  • チャット reply length / timeout — 低速なハードウェア向けにトークン数と待ち時間を制限します。
  • バイナリディレクトリ / モデルディレクトリ — 既定パスを上書きします。パスをデフォルトにリセット で同梱レイアウトに戻せます。
Moni 管理のサーバーは localhost の 8081 から 8299 までのうち、最初に空いているポートへバインドされます。ポートはプロセス識別情報とともに保存されるため、Moni は再起動後に再接続できます。

Ollama

実行中の Ollama インスタンス（既定の localhost base URL）を Moni に指定します。Ollama が公開している一覧からモデルを選択します。チャットを始める前に、Ollama がインストール済みで、モデルが取得済み（ollama pull …）であることを確認してください。

クラウドプロバイダー (OpenAI, Claude, Gemini)

API キーを入力します（ローカル設定に保存されます）。パネルに表示されるモデルや、各プロバイダー固有のオプションを選択できます。クラウドチャットはローカル GPU を必要としませんが、会話内容をベンダーの API に送るため、外部共有できない journal には使用しないでください。

関連する hledger 設定（設定 → hledger 配下）: AI 返信の hledger コマンドを検証 は、Run を押す前に、バージョン一致の complist とあなたのバイナリに照らしてアシスタントの例をチェックします。

AI Wrapper（実験的）

同じマシン上で動作する Runable AI Wrapper デスクトップアプリに接続します。AI Wrapper → API でクライアントを登録し、Preferences でサービスを有効化してから、登録トークンを Moni に貼り付けます。サービス、モード（Electron のアプリ内 webview または Chrome 拡張）、クライアントラベルを選択し、接続 をクリックして利用可能なサービスを一覧表示します。

コンパニオンアプリが未インストールの場合、AI Wrapper へのアクセスについて Runable.app にお問い合わせください。

hledger ドキュメント (RAG)

Moni は hledger ドキュメントを埋め込むことで、ローカル AI の回答が正しいフラグや概念を参照できるようにします。AI パネルのステータス行には次が表示されます:

  • 利用不可（ローカル AI では任意）。 — ローカル AI では任意です。クラウドはこれがなくても動作する場合があります。
  • ステータス: ディスク上のファイル。最初の AI メッセージ時にメモリに読み込まれます（すべてのプロバイダー）。 — ドキュメント索引が存在し、最初のメッセージ送信時に読み込まれます。
  • 準備完了 — N 個のドキュメントチャンク。 — 根拠付き応答のための検索が有効です。

パス、検索順序、complist の詳細は、設定 → ログ → デバッグ の app.log、または AI パネルのプロバイダー 📜 ログ ボタンを確認してください。

メイン でのチャット動作

プロバイダーを接続したら、メイン → チャット に入力します。Moni は実行可能な hledger コマンドを提案することがあり、Enter を押すかコントロールを使って 台帳 パネルで実行できます。自動補完（設定 → hledger）が有効な場合、hledger argv を入力すると候補ポップオーバーが表示され、Ctrl+H で入力行から hledger を実行できます。

インタラクションログ（設定 → ログ）では、ユーザー入力、AI 会話、台帳パネルの結果を、自分用の監査記録として保存できます。

トラブルシューティング

  • OpenAI 互換 API に接続できません。 — llama.cpp の場合は サーバーを起動 が成功したか確認してください。診断 LED からサーバーログを開けます。localhost がファイアウォールで遮断されていないことも確認してください。
  • Wrong GPU build — CPU 専用バイナリでは GPU トグルは使えません。GPU 対応の llama-server 版をダウンロードまたはインストールしてください。
  • Empty or “thinking” replies — モデルの推論を無効化 を有効にして、サーバーを再起動してください。
  • Cloud errors — API キー、モデル名、ネットワークを確認し、HTTP ステータスは 📜 ログ で確認してください。
  • RAG not loading — app.log のデバッグ行を確認してください。索引の読み込み中は最初のチャットが長くかかる場合があります。

問題を報告する際は、llama.cpp のdiagnostics bundle（状態 + ログ末尾）をコピーしてください。journal を公開せずに、サポートに必要な情報を一通り含められます。
