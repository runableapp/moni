設定

設定 は、左側にセクション一覧、右側に詳細パネルがあるフルタブとして開きます。まず hledger を設定し、その後に口座、ツールバーボタン、必要に応じて AI を設定してください。多くのセクションはユーザー設定フォルダ（Linux では ~/.config/moni/）に保存されます。

一般

UI 言語、起動時の動作、そのほかアプリ全体の設定です。言語を変更すると、翻訳があるタブ、モーダル、ヘルプ トピックにわたってローカライズされた文字列が再読み込みされます。

AI

メイン → チャット で使う AI プロバイダーを選択します。ローカルの llama.cpp または Ollama（データは自分のマシン内に留まります）、あるいはクラウドの OpenAI、Claude、Gemini を利用できます。任意の AI Wrapper は、拡張ベースのサービス向けに Runable デスクトップラッパーへ接続します。

ローカル AI には、バイナリ/モデルのダウンロード、サーバーの起動/停止、GPU オプション、そして根拠ある回答のための hledger ドキュメント RAG（retrieval-augmented generation）が含まれます。

詳細手順: 設定 → AI

口座

hledger から口座ツリーを更新し、その後 メイン と 口座 に表示する口座を選びます。表示プロファイルとして、金融機関名、マスクされた口座ヒント、アイコン、並び順も設定できます。これらのプロファイルは merge_account_profiles のような YAML 変換でも使われます。

MCP

対応する AI 構成で Moni が利用できる Model Context Protocol サーバー参照を設定します。組み込みの hledger ヘルプを超えてチャット機能を広げるツールのために、サーバー定義や接続情報を追加します。

詳細設定

上級者向けオプションです。カスタム設定パス、実験的フラグ、開発者向けトグルが含まれます。journal への書き込みや AI の挙動に与える影響を理解している場合にのみ変更してください。

AI エージェント

Moni がエージェントプロファイル経由でチャットをルーティングするときに使うエージェント定義（システムプロンプト、ツールアクセス）を管理します。記帳向けと一般質問向けで別々の役割を用意したいときに便利です。

ボタン

ツールバーボタンを有効にし、各行の yamlRef にレシピ id または YAML ファイルパスを設定します。行の並び替えで メイン のツールバー配置を変更できます。レシピの作成方法は カスタムボタン (YAML) を参照してください。

外観

テーマ（light/dark/system）とフォントサイズです。フォントサイズは表、チャット、取引先レジスターダイアログ全体に適用され、読みやすさを統一します。

バックアップ

Moni が journal ファイルを変更する前に、自動で journal をバックアップします。file backup（journal の横、または指定ディレクトリにタイムスタンプ付きコピーを作成）および/または git backup（リポジトリへコミット）を有効にできます。バックアップ now で手動スナップショットを作成し、必要に応じてファイルまたは git 履歴から復元できます。

hledger

必須です。hledger バイナリのパスとメイン journal ファイル（-f）を設定します。Moni は残高、レジスター、インポート、ボタンレシピの実行に hledger を使います。任意のトグルとして、台帳 結果の下にコピー&ペースト用 CLI を表示する、チャットで hledger を自動補完する、AI が提案したコマンドをインストール済みバージョンと照合して検証する、などがあります。

インストールされていない場合は、ダウンロード補助でプラットフォーム向けのビルド済み hledger リリースを取得できます。

インポート

CSV インポートルールのディレクトリ、ファイル名とルールの対応、スマートインポート トグル、ドロップ時 dry-run、重複チェック、アーカイブディレクトリ、重複バックアップの挙動を設定します。Moni はまずルールディレクトリで bank.csv.rules を探し、その後 CSV ファイルの隣を探します。

ログ

Moni は interaction.log（チャットと台帳パネル）および app.log（起動、エラー、デバッグ）を書き込みます。ログディレクトリの設定、記録する操作イベントの選択、app.log のデバッグ有効化、フォルダを開く、ログを削除することができます。AI 接続や YAML ボタンの失敗を調査するときに役立ちます。
