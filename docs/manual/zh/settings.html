设置

设置 会以完整标签页形式打开，左侧是分区列表，右侧是详细面板。请先配置 hledger，然后再配置账户、工具栏按钮和可选的 AI。大多数分区会保存到你的用户配置目录（Linux 上为 ~/.config/moni/）。

常规

UI 语言、启动行为以及其他应用级偏好设置。更改语言后，所有已提供翻译的标签页、模态框和 帮助 主题中的本地化字符串都会重新加载。

AI

为 主页 → 聊天 选择一个 AI 提供商：本地 llama.cpp 或 Ollama（数据保留在你的机器上），或云端 OpenAI、Claude、Gemini。可选的 AI Wrapper 可连接到 Runable 桌面包装器，以使用基于扩展的服务。

本地 AI 包含二进制文件/模型下载、服务器启动/停止、GPU 选项，以及用于获得有依据回答的 hledger 文档 RAG（retrieval-augmented generation）。

完整步骤说明：设置 → AI

账户

从 hledger 刷新账户树，然后选择哪些账户显示在 主页 和 账户 中。你还可以添加显示资料：机构名称、打码后的账户提示、图标和排序顺序。这些资料也会用于 merge_account_profiles 等 YAML 转换。

MCP

配置 Moni 在受支持 AI 设置中可使用的 Model Context Protocol 服务器引用。你可以添加服务器定义和连接详情，用于扩展聊天能力，使其超越内置的 hledger 帮助。

高级

面向高级用户的选项：自定义配置路径、实验性标志和开发者开关。只有在你了解其对 journal 写入和 AI 行为的影响时才应修改这些设置。

AI 代理

管理 Moni 通过代理资料路由聊天时使用的代理定义（系统提示词、工具访问权限）。如果你希望记账问题和一般问题使用不同的角色设定，这会很有帮助。

按钮

启用工具栏按钮，并将每一行的 yamlRef 设为配方 id 或 YAML 文件路径。你可以重新排序行，以改变 主页 上的工具栏布局。关于配方编写，请参见 自定义按钮 (YAML)。

外观

主题（light/dark/system）和字体大小。字体大小会统一应用于表格、聊天和交易方寄存器对话框，以保持一致的可读性。

备份

Moni 在修改 journal 文件前会自动创建 journal 备份。可启用 file backup（在 journal 旁边或指定目录创建带时间戳的副本）和/或 git backup（在仓库中提交）。使用 备份 now 可手动创建快照；需要时可从文件或 git 历史中恢复。

hledger

必需。设置 hledger 二进制文件路径和主 journal 文件（-f）。Moni 运行余额、寄存器、导入和按钮配方时都会调用 hledger。可选开关包括：在 账本 结果下显示可复制粘贴的 CLI、聊天中的 hledger 自动补全，以及将 AI 建议的命令与你已安装的版本进行校验。

如果系统中尚未安装，还可以使用下载助手获取适用于你平台的预构建 hledger 发行版。

导入

CSV 导入规则目录、文件名到规则的映射、智能导入 开关、拖放时 dry-run、重复检查、归档目录以及重叠备份行为。Moni 会先在规则目录中查找 bank.csv.rules，然后再查找 CSV 文件旁边的同名规则。

日志

Moni 会写入 interaction.log（聊天和账本面板）以及 app.log（启动、错误、调试）。你可以配置日志目录、选择要记录的交互事件、启用 app.log 调试、打开文件夹或删除日志。在排查 AI 连接问题或 YAML 按钮失败时非常有用。
