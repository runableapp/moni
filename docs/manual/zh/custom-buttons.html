自定义按钮 (YAML)

工具栏和账本工具由YAML 配方定义。设置 → 按钮 中的每个自定义按钮都通过 yamlRef（配方 id 或文件路径）指向一个配方。Moni 会加载 YAML，运行 hledger 或外部命令，并将结果渲染到 账本 面板中（表格或等宽文本）。

配方存放位置

位置用途
~/.config/moni/custom_buttons/<id>.yaml用户覆盖项（按 id 查找时优先级最高）
<executable>/custom_buttons/<id>.yaml随 Moni 二进制文件一起提供
任何以 .yaml / .yml 结尾，或包含 / 的路径被引用时直接加载

在 设置 → 按钮 中，每一行都有一个 yamlRef：可以是配方 id（例如 journal_check），也可以是绝对/相对路径（例如 ~/recipes/my_tool.yaml）。配方 id 必须符合 ^[a-zA-Z0-9][a-zA-Z0-9._-]*$（不能包含 .. 或斜杠）。

随应用提供的示例位于应用的 custom_buttons/ 文件夹。你可以把其中一个复制到自己的配置目录中进行自定义，而无需重新构建 Moni。

最小配方：一个 hledger 命令

hledger check 在 journal 健康时不会产生任何输出。你可以显示一条友好的消息，而不是空白面板：

id: journal_check

meta:
  name: Journal check
  description: Run hledger check on the full journal.

run:
  mode: hledger
  argv:
    - check

output:
  plain_text: true
  when_empty: "Everything is ok! No issues reported by hledger check."
  • run.mode: hledger — Moni 会根据 设置 → hledger 运行 hledger -f <your journal> …。
  • run.argv — 是 -f <journal> 之后的参数，而不是完整的 shell 命令行。
  • output.plain_text: true — 以等宽文本显示 stdout/stderr。
  • output.when_empty — 如果显示结果为空，则显示这段文字。

将按行输出解析为带样式的表格

hledger stats 会输出 key: value 形式的行。可以将其转换为两列表格：

id: journal_stats

meta:
  name: Journal overview
  description: hledger stats summary.

run:
  mode: hledger
  argv:
    - stats

output:
  delimited_text:
    delimiter: ":"
    hide_header: true
    no_striping: true
    column_align: [right, left]
    column_widths: [30%, 1fr]
    column_borders: true
    column_styles:
      - background: accent-24
        bold: true
      - {}

output.delimited_text 会在每个非空行首次出现 delimiter 的位置进行分割。除非你设置了 columns，否则默认列标题是 Field / Value。

多步骤配方：运行 hledger、解析 CSV、显示表格

大多数账本工具都使用 steps：每一步都有一个 id，运行 hledger，并可选择解析或转换结果。

id: register_recent

meta:
  name: Recent activity
  description: Register for a date range.

ui:
  modal:
    when_empty:
      period: ytd
    fields:
      - id: date_from
        type: date
        label_key: common.dateFrom
        optional: true
      - id: date_to
        type: date
        label_key: common.dateTo
        optional: true

steps:
  - id: reg
    run:
      argv:
        - register
        - -p
        - "{{period}}"
        - -O
        - csv
    parse:
      kind: hledger_register_csv

output:
  from_step: reg
  table_view: register

流程：用户点击按钮 → 模态框收集日期 → Moni 构建 {{period}} → 步骤运行 hledger … register -p <period> -O csv → 解析器构建列 → output.from_step 选择该表 → table_view: register 启用寄存器样式下钻。

模态框留空时的默认日期

ui:
  modal:
    when_empty:
      period: lastyear
    fields:
      - id: date_from
        type: date
        optional: true
      - id: date_to
        type: date
        optional: true

如果配方使用 {{period}}，且用户将两个日期都留空，则会应用 when_empty.period。用户输入的日期始终优先。

令牌范围
lastyear上一日历年
thisyear当前日历年
thismonth本月第一天 .. 今天
lastmonth上一日历月
last30days今天 − 30 天 .. 今天
ytd, yeartodate1 月 1 日 .. 今天

转换：重塑 balance CSV

常见模式是：run → parse hledger_balance_csv → transform → output。

steps:
  - id: csv
    run:
      argv:
        - balance
        - "^expenses:"
        - -p
        - "{{period}}"
        - --flat
        - -O
        - csv
    parse:
      kind: hledger_balance_csv
  - id: table
    transform:
      kind: flat_balance_table
      source_step: csv

output:
  from_step: table

merge_account_profiles 会通过 设置 中的资料显示名称映射账户名。stacked_balance_reports 则会把两个 balance CSV 步骤合并为一个带分节标题的双列表格报表。

一个结果中包含多个表格

分离的分节（例如资产负债表）：

output:
  sections:
    - title: Assets
      from_step: assets_table
    - title: Liabilities
      from_step: liab_table

合并比较（每个账户一行，每个期间一列）：

output:
  merge_balance_sections: true
  sections:
    - title: "{{period_a}}"
      from_step: table_a
    - title: "{{period_b}}"
      from_step: table_b

可以在某个分节上添加 when_empty，以便在该分节没有行时显示占位消息。

外部程序（不是 hledger）

可执行文件

argv[0] 是程序，后续条目是参数。运行前会先将模态值替换进 argv。

run:
  mode: executable
  argv:
    - /usr/bin/cal
    - "{{month}}"
    - "{{year}}"

output:
  plain_text: true

Shell

将整个脚本作为一个字符串（Unix: sh -c；Windows: cmd /C）：

run:
  mode: shell
  script: |
    echo "Journal: $HLEDGER_JOURNAL"
    wc -l "$HLEDGER_JOURNAL"

环境变量包括 HLEDGER_JOURNAL、HLEDGER_JOURNAL_FILE 和 MONI_TOOLS_DIR。

输出后处理：管道与空消息

output:
  plain_text: true
  pipe:
    argv: ["grep", "-i", "error"]

output.when_empty 适用于经过管道处理后的整个结果。对于没有任何行的多步骤表格，它会显示你的消息，而不是空白网格。

argv 中的占位符会先被替换，然后空的 -b、-e、-p、-f 及其值会被移除，因此未使用时可选的期间/文件标志会自动消失。

内置查询辅助项

如果 YAML 中包含这些占位符，Moni 会自动填充：

占位符为空时会变成
{{expense_query}}^expenses 或来自模态 expense_leaf 的 expenses:<leaf>
{{balance_query}}^income 或来自模态 customer 的 income:consulting:<customer>
{{tag_slice_query}}^expenses: 或来自模态 tag 的 expenses tag:<tag>

可选图表（pie、bar、line）

在模态框中添加图表字段，并在 YAML 中配置 output.chart。标准模态字段如下：

ui:
  modal:
    when_empty:
      chart_display: "off"
      chart_period: monthly
    fields:
      - id: chart_display
        type: select
        optional: true
        options: ["off", "pie", "bar", "line"]
      - id: chart_period
        type: select
        optional: true
        options: [daily, monthly, yearly]

output:
  chart:
    param: chart_display
    period:
      param: chart_period
      argv_param: period_flag
      when_kinds: [bar, line]
      flags: { daily: "-D", monthly: "-M", yearly: "-Y" }
    layouts:
      bar: stacked_period
      line: period_series
      pie: category_totals

在 设置 中接入

  1. 将配方保存为 ~/.config/moni/custom_buttons/my_tool.yaml，并确保顶层 id: 匹配。
  2. 打开 设置 → 按钮 → 添加或编辑一行。
  3. 将 yamlRef 设为 id（例如 my_tool）或完整路径。
  4. 启用按钮并返回 主页，它就会出现在工具栏中。
应用安装包中自带的配方是一个不错的起点：复制一个到你的配置文件夹中，编辑后让 yamlRef 指向你的副本。
