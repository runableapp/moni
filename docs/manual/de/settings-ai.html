Einstellungen → KI

Übersicht → Chat uses the provider selected here. Local-first options (llama.cpp, Ollama) keep your journal and messages on your machine. Cloud providers send chat to their APIs — use them only if you accept that trade-off. Choose Keiner to disable chat until you are ready.

After changing provider, use the connectivity check (status indicator) and 📜 Protokolle button in the KI panel if chat fails to respond.

Local KI (llama.cpp)

Moni can download a llama-server binary and a GGUF model, then manage the server process on 127.0.0.1 (ports 8081–8299). Recommended first-time flow: click Lokale KI einrichten or download binary + model separately, then Server starten.

Key controls

  • Zu verwendendes Modell — pick a bundled preset or choose another GGUF file. Changing model while the server runs may prompt to restart.
  • GPU verwenden, falls verfügbar — requires a GPU-capable llama-server build; pick a specific GPU device or “All GPUs”.
  • Modell-Reasoning deaktivieren — recommended for chat; passes --reasoning off so replies appear in the panel (needed for Qwen 3 and similar “thinking” models).
  • Modelle ohne Neustart wechseln (Router-Modus) — swap models without full restart when using a multi-model --models-dir layout.
  • Lokale KI beim Start automatisch starten — starts the managed server after the app loads (binary and model must already be installed).
  • Chat reply length / timeout — cap tokens and wait time for slow hardware.
  • Programmverzeichnis / Modellverzeichnis — override default paths; Pfade auf Standard zurücksetzen restores shipped layout.
Moni-managed server binds the first free port on localhost from 8081 to 8299. The port is saved with the process identity so Moni reconnects after restart.

Ollama

Point Moni at your running Ollama instance (default base URL on localhost). Select a model from the list Ollama exposes. Ensure Ollama is installed and the model is pulled (ollama pull …) before chatting.

Cloud providers (OpenKI, Claude, Gemini)

Enter your API key (stored in local config). Pick model and any provider-specific options shown in the panel. Cloud chat does not require a local GPU but sends conversation content to the vendor’s API — do not use for journals you cannot share externally.

Related hledger settings (under Einstellungen → hledger): hledger-Befehle in KI-Antworten prüfen lint assistant examples against the version-matched complist and your binary before you click Run.

KI Wrapper (experimental)

Verbindens to the Runable KI Wrapper desktop app on the same machine. Register a client in KI Wrapper → API, enable services in Preferences, then paste the registration token in Moni. Choose service, mode (Electron in-app webview or Chrome-Erweiterung), and client label. Click Verbinden to list available services.

Contact Runable.app for KI Wrapper access if you do not have the companion app installed.

hledger-Dokumentationsindex (RAG)

Moni can embed hledger documentation so local KI answers cite correct flags and concepts. Status lines in the KI panel show:

  • Nicht verfügbar — optional for local KI; cloud may still work without it.
  • Auf Disk; lädt beim ersten Chat — doc index present, loads when you first send a message.
  • Bereit — N Dokumentabschnitte — retrieval is active for grounded replies.

For paths, search order, and complist details, check Einstellungen → Protokolle → Debug in app.log, or the provider 📜 Protokolle button in the KI panel.

Chat behavior on Übersicht

With a provider connected, type in Übersicht → Chat. Moni may suggest runnable hledger commands; press Enter or use controls to execute them in the Ledger-Panel-Ergebnisse. With autocomplete enabled (Einstellungen → hledger), typing hledger argv shows suggestion popover; Ctrl+H runs hledger from the input line.

Interaction logging (Einstellungen → Protokolle) can record user input, KI conversation, and ledger panel results for your own audit trail.

Troubleshooting

  • OpenKI-kompatible API ist nicht erreichbar. — for llama.cpp, confirm Server starten succeeded; open server log from the diagnostics LED. Check firewall is not blocking localhost.
  • Wrong GPU build — CPU-only binary cannot use GPU toggle; download or install a GPU-capable llama-server variant.
  • Empty or “thinking” replies — enable Modell-Reasoning deaktivieren and restart the server.
  • Cloud errors — verify API key, model name, and network; read 📜 Protokolle for HTTP status.
  • RAG not loading — see app.log debug lines; first chat may take longer while the index loads.

Copy the llama.cpp diagnostics bundle (status + log tail) when reporting issues — it includes everything support needs without exposing your journal.
