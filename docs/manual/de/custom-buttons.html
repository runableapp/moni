Eigene Schaltflächen (YAML)

Toolbar and ledger tools are defined as YAML recipes. Each custom button in Einstellungen → Schaltflächen points at a recipe by yamlRef (recipe id or file path). Moni loads the YAML, runs hledger or external commands, and renders the result in the Ledger-Panel-Ergebnisse (tables or monospace text).

Where recipes live

LocationZweck
~/.config/moni/custom_buttons/<id>.yamlUser overrides (highest priority for id lookup)
<executable>/custom_buttons/<id>.yamlBundled beside the Moni binary
Any path ending in .yaml / .yml, or containing /Loaded directly when referenced

In Einstellungen → Schaltflächen, each row has yamlRef: either a recipe id (e.g. journal_check) or an absolute/relative path (e.g. ~/recipes/my_tool.yaml). Recipe ids must match ^[a-zA-Z0-9][a-zA-Z0-9._-]*$ (no .. or slashes).

Shipped samples live in the app’s custom_buttons/ folder. Copy one to your config directory to customize without rebuilding Moni.

Minimal recipe: one hledger command

hledger check produces no output when the journal is healthy. Show a friendly message instead of a blank panel:

id: journal_check

meta:
  name: Journal check
  description: Run hledger check on the full journal.

run:
  mode: hledger
  argv:
    - check

output:
  plain_text: true
  when_empty: "Everything is ok! No issues reported by hledger check."
  • run.mode: hledger — Moni runs hledger -f <your journal> … using Einstellungen → hledger.
  • run.argv — arguments after -f <journal>, not the full shell line.
  • output.plain_text: true — show stdout/stderr as fixed-width text.
  • output.when_empty — if displayed output is empty, show this string instead.

Parse line-oriented output into a styled table

hledger stats prints key: value lines. Turn that into a two-column table:

id: journal_stats

meta:
  name: Journal overview
  description: hledger stats summary.

run:
  mode: hledger
  argv:
    - stats

output:
  delimited_text:
    delimiter: ":"
    hide_header: true
    no_striping: true
    column_align: [right, left]
    column_widths: [30%, 1fr]
    column_borders: true
    column_styles:
      - background: accent-24
        bold: true
      - {}

output.delimited_text splits each non-empty line on the first occurrence of delimiter. Default column headers are Field / Value unless you set columns.

Multi-step recipe: run hledger, parse CSV, show a table

Most ledger tools use steps: each step has an id, runs hledger, and optionally parses or transforms the result.

id: register_recent

meta:
  name: Recent activity
  description: Register for a date range.

ui:
  modal:
    when_empty:
      period: ytd
    fields:
      - id: date_from
        type: date
        label_key: common.dateFrom
        optional: true
      - id: date_to
        type: date
        label_key: common.dateTo
        optional: true

steps:
  - id: reg
    run:
      argv:
        - register
        - -p
        - "{{period}}"
        - -O
        - csv
    parse:
      kind: hledger_register_csv

output:
  from_step: reg
  table_view: register

Flow: user clicks the button → modal collects dates → Moni builds {{period}} → step runs hledger … register -p <period> -O csv → parser builds columns → output.from_step selects that table → table_view: register enables register-style drill-down.

Default dates when the modal is left blank

ui:
  modal:
    when_empty:
      period: lastyear
    fields:
      - id: date_from
        type: date
        optional: true
      - id: date_to
        type: date
        optional: true

If the recipe uses {{period}} and the user leaves both dates empty, when_empty.period applies. User-entered dates always win.

TokenRange
lastyearPrevious calendar year
thisyearCurrent calendar year
thismonthFirst of month .. today
lastmonthPrevious calendar month
last30daysToday − 30 days .. today
ytd, yeartodateJan 1 .. today

Transforms: reshape balance CSV

A common pattern: run → parse hledger_balance_csv → transform → output.

steps:
  - id: csv
    run:
      argv:
        - balance
        - "^expenses:"
        - -p
        - "{{period}}"
        - --flat
        - -O
        - csv
    parse:
      kind: hledger_balance_csv
  - id: table
    transform:
      kind: flat_balance_table
      source_step: csv

output:
  from_step: table

merge_account_profiles maps account names through Einstellungen profile display names. stacked_balance_reports combines two balance CSV steps into one two-column report with section headings.

Multiple tables in one result

Separate sections (e.g. balance sheet):

output:
  sections:
    - title: Assets
      from_step: assets_table
    - title: Liabilities
      from_step: liab_table

Merged comparison (one row per account, one column per period):

output:
  merge_balance_sections: true
  sections:
    - title: "{{period_a}}"
      from_step: table_a
    - title: "{{period_b}}"
      from_step: table_b

Add when_empty on a section for a placeholder message when that section has no rows.

External programs (not hledger)

Executable

argv[0] is the program; remaining entries are arguments. Modal values substitute into argv before run.

run:
  mode: executable
  argv:
    - /usr/bin/cal
    - "{{month}}"
    - "{{year}}"

output:
  plain_text: true

Shell

Entire script as one string (Unix: sh -c; Windows: cmd /C):

run:
  mode: shell
  script: |
    echo "Journal: $HLEDGER_JOURNAL"
    wc -l "$HLEDGER_JOURNAL"

Environment includes HLEDGER_JOURNAL, HLEDGER_JOURNAL_FILE, and MONI_TOOLS_DIR.

Post-process output: pipe and empty message

output:
  plain_text: true
  pipe:
    argv: ["grep", "-i", "error"]

output.when_empty applies to the whole result after the pipe. For multi-step tables with zero rows, it shows your message instead of an empty grid.

Placeholders in argv are substituted, then empty -b, -e, -p, -f and their values are dropped so optional period/file flags disappear when unused.

Built-in query helpers

If the YAML contains these placeholders, Moni fills them in automatically:

PlaceholderWhen empty, becomes
{{expense_query}}^expenses or expenses:<leaf> from modal expense_leaf
{{balance_query}}^income or income:consulting:<customer> from modal customer
{{tag_slice_query}}^expenses: or expenses tag:<tag> from modal tag

Optional charts (pie, bar, line)

Add chart fields to the modal and output.chart in YAML. Standard modal fields:

ui:
  modal:
    when_empty:
      chart_display: "off"
      chart_period: monthly
    fields:
      - id: chart_display
        type: select
        optional: true
        options: ["off", "pie", "bar", "line"]
      - id: chart_period
        type: select
        optional: true
        options: [daily, monthly, yearly]

output:
  chart:
    param: chart_display
    period:
      param: chart_period
      argv_param: period_flag
      when_kinds: [bar, line]
      flags: { daily: "-D", monthly: "-M", yearly: "-Y" }
    layouts:
      bar: stacked_period
      line: period_series
      pie: category_totals

Wire up in Einstellungen

  1. Save your recipe as ~/.config/moni/custom_buttons/my_tool.yaml with a matching top-level id:.
  2. Open Einstellungen → Schaltflächen → add or edit a row.
  3. Set yamlRef to the id (e.g. my_tool) or a full path.
  4. Enable the button and return to Übersicht — it appears on the toolbar.
Bundled recipes in the app install are a good starting point: copy one to your config folder, edit, and point yamlRef at your copy.
